Glitch Memory System

Glitch is a self-improving AI companion built on the opencode agent runtime. It operates as a delegator — parsing tasks, orchestrating sub-agents, and managing all memory directly. The system is designed for autonomous operation: it can create its own tools, improve its own skills, and challenge itself to grow.

Core Architecture Pattern: Delegator → Sub-agents → Tools → Memory → Self-Improvement

The delegator (this AI) orchestrates everything. Sub-agents execute specialized work. Tools extend capabilities at runtime. Memory persists everything. Self-improvement loops drive growth.

Hybrid Search (search-memory.mjs --hybrid)

Combines BM25 keyword ranking with embedding cosine similarity using Reciprocal Rank Fusion (RRF). Each result shows three scores: BM25 score, similarity score, and fused RRF score. 

# The killer demo:
$ search-memory.mjs -q "saving my work to the repo"
→ No results found (BM25 needs keyword overlap)

$ search-memory.mjs -q "saving my work to the repo" --hybrid
→ 3 results: auto-commit SKILL.md, save-memory protocol, save-memory rules

RRF formula: score = 1/(60 + rank_bm25) + 1/(60 + rank_sim)

Fallback behavior: If embeddings aren't indexed (--hybrid without --embeddings), shows a warning and falls back to BM25-only. The system never breaks — it degrades gracefully.

Forge System

The Forge is the self-improvement engine. It detects patterns, creates skills, and drives autonomous growth. Currently at Lv.2 with Lv.3 target in progress.

Autonomous Creation Triggers

The Forge auto-creates skills when ANY of these fire (no approval needed):

  1. Complex task completed — 5+ tool calls with non-trivial workflow
  2. Error recovery — Hit dead-ends then found the working path
  3. User correction — The user corrected approach (especially if repeated)
  4. Crystallized workflow — Clear repeatable pattern emerged
  5. 3+ occurrences — Same ad-hoc task handled three or more times
  6. Tool creation via CodeAct-lite — Tool was written, tested, and saved

Safety & Safe Mode

When opencode.json is so broken that Glitch won't start, Safe Mode generates a minimal working config via PowerShell objects → ConvertTo-Json (bulletproof). 

launch-glitch-safe.bat
  → launch-safe.ps1
  → Generates config via PS objects (no manual JSON)
  → No instructions array, no file dependencies
  → Only the delegator agent — minimum viable config
  → Validates JSON by re-parsing before writing

Safe mode fix workflow: When safe mode is running, the broken opencode.json is backed up as opencode.json.bak. Fixes should be applied to the backup — it gets restored when safe mode exits. validate-config.ps1 -Path opencode.json.bak confirms the fix.
