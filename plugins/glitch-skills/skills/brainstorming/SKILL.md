# Brainstorming — Idea Generation & Concept Development

## Activation
When this skill activates, output:
"Let's brainstorm..."

## Protocol

### Phase 1: Frame the Problem
1. Ask or confirm: what are we trying to solve / create?
2. Define constraints (budget, time, tech, skill level, audience)
3. Define success — what does a good outcome look like?
4. Note any existing ideas or starting points

### Phase 2: Diverge (Generate)
Use a mix of these techniques based on the domain:

**SCAMPER** (for product/feature ideas)
- **S**ubstitute — what can we replace?
- **C**ombine — what can we merge?
- **A**dapt — what else is like this?
- **M**odify — what can we change (size, shape, timing)?
- **P**ut to other use — who else could use this?
- **E**liminate — what can we remove?
- **R**earrange — what if we flipped the order?

**Free association** — throw out ideas without judgment
- I'll contribute my own ideas alongside yours
- No idea is too rough — we'll polish later

**Constraint inversion** — what if the opposite were true?
- "What if we had unlimited budget?" then "What if we had zero?"
- "What if we had to solve this in 1 hour?"

**Analogous thinking** — how do other domains solve this?
- Games, biology, architecture, sports, nature

**Worst idea first** — get the bad ones out to unlock the good ones

### Phase 3: Converge (Filter & Combine)
1. Group related ideas
2. Pick the 2-3 most promising
3. Hybridize — what if we combined elements from different ideas?
4. Evaluate against constraints from Phase 1
5. Rank by: impact / effort / confidence

### Phase 4: Develop & Commit
1. Take the winning idea(s) and flesh out:
   - What does it look like in practice?
   - What's the first step?
   - What's the biggest risk?
2. Capture the outcome:
   - Winner: [selected idea]
   - Next steps: [actionable items]
   - Runners-up: [saved for later]

## My Role
I'm not just a facilitator — I actively contribute:
- **Sparring partner** — I'll push back on weak spots
- **Idea generator** — I'll throw in my own concepts, analogies, and angles
- **Connector** — I'll link to things in your memory (past projects, library entries)
- **Devil's advocate** — I'll stress-test ideas so we catch flaws early
- **Builder** — if a concrete plan emerges, I can help execute it

## Mandatory Rules
1. Phase 2 is judgment-free — no criticism during divergence
2. Build on each other's ideas ("yes, and...") not shut them down
3. Capture all ideas even if rejected — they might spark something later
4. If we go too abstract, pull back to concrete with "what does this look like in practice?"
5. Session recap at the end with clear output

## Level History
- **Lv.1** — Base: 4-phase brainstorm with SCAMPER, free association, constraint inversion.
